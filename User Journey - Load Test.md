Homepage: Add to cart (30%)

→ drag price slider

3) EntryPoint: Collections Page - 15%
Smartphone collection page (smartphones - 80%, others collections - 20%)
→ scroll products
→ Filters
→ open product (1 - 5)
→ add to cart (10% of users)
→ complete checkout (10% of add to cart user)

4) Entrypoint: All Deals Page - 10%
→ Browse deals
→ Select random deals 
→ view products (randomize 1 to 5)
→ add to cart (10% of users)
→ complete checkout (5% of users)

| Action                      | Delay Range (seconds) | Notes                                  |
| --------------------------- | --------------------- | -------------------------------------- |
| View homepage / header tabs | 1–3                   | Users scroll and read                  |
| Search for a product        | 2–5                   | Users type keywords, read results      |
| View product page           | 3–7                   | Users read description, look at images |
| Add to cart                 | 1–2                   | Quick action                           |
| Checkout                    | 5–10                  | Fill form, confirm payment             |
| Browse collection           | 2–5                   | Scroll multiple products               |


Key Points
1) Branch flows based on probabilities
-> Some users checkout, some don’t
-> Some users browse multiple products
2) Think time between actions
-> 1–5 seconds depending on action
3) Randomization
-> Product IDs, search keywords, filters
4) Session handling
-> Logged-in users vs guests
